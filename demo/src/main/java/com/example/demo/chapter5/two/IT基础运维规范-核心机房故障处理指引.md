IT基础运维规范-核心机房故障处理指引
1. 适用范围
   本规范适用于公司总部A区与B区所有核心机房的服务器物理故障排查与应急响应。

2. 戴尔R740服务器常见报警与处理流程
   当监控系统捕获到戴尔R740服务器出现硬件级别异常时，驻场运维工程师必须严格按照以下步骤进行操作：

2.1 闪黄灯报警处理
如果机箱前面板出现闪黄灯报警：

严禁在未确认故障源的情况下直接拔插电源。

登录带外管理接口（iDRAC），查看“系统健康状况”和“生命周期日志”。

若确认为内存ECC错误或阵列卡电池老化，需在内部工单系统中提交“硬件更换申请”。

2.2 iDRAC重置与安全审批流程
当服务器发生假死，且无法通过网络访问iDRAC管理界面时，需要执行物理重置或硬重启。
安全审批流程要求：

工程师需在OA系统中发起《核心机房高危操作审批单》。

审批单必须明确包含：故障服务器资产编号、受影响的业务系统、预计中断时间。

该审批单必须依次经过IT运维总监和业务部门技术BP的双重电子签名授权。

审批通过后，方可长按服务器前面板的“i」”按钮（系统识别按钮）15秒以上强制重置iDRAC。未获审批的强制重启将触发安全违规通报。
